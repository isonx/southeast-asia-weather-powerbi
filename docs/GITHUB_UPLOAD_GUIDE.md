## Upload the prepared repository

1. Create a new empty GitHub repository.
2. Extract the repository ZIP supplied with this project.
3. Upload the **contents of the extracted folder**, not the outer ZIP itself.
4. Keep the folder structure unchanged so the README image and documentation links work.
5. Commit the files to the `main` branch.
6. Open the repository page and verify:
   - The dashboard preview appears at the top of the README.
   - The PBIX download link works.
   - Every file under `docs/` opens correctly.

## Optional GitHub Release

The PBIX is small enough to remain in this repository. You can also create a versioned release:

- Tag: `v1.0.0`
- Title: `Southeast Asia Weather Intelligence v1.0.0`
- Asset: `SEAWeatherIntelligence.pbix`

A release makes it easier for visitors to find a stable downloadable version.

Do not remove the Open-Meteo attribution from the README or NOTICE file. Do not commit Power BI cache files or credentials.
